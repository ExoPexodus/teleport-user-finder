# UI Wireframes Document

## Overview

This document provides wireframes and UI specifications for the Teleport User Management System. The application follows an **enterprise-grade design language** with a collapsible sidebar navigation, table-first user list, and wizard-style workflows.

---

## Design Philosophy

- **Calm & Confident**: Enterprise tool vibes, minimalist but informative
- **Table-First**: Admins prefer tables for high-density data scanning
- **Smart Filters**: Collapsible with chip-based active filter summary
- **Guided Workflows**: Step indicators for complex operations
- **Incident Handling**: Serious UI for dangerous operations

|-------|-------|-------|

| `--secondary` | Secondary buttons | Muted gray |

| `--muted` | Disabled states | Light gray |
| `--accent` | Highlights | Soft indigo |
| `--destructive` | Delete actions | Red |

| `--status-warning` | Inactive status | Amber |

| `--portal-kocharsoft` | Portal badge | Blue |

| Token | Usage |
|-------|-------|
| `--shadow-sm` | Cards, buttons |
| `--shadow-md` | Dropdowns, dialogs |
| `--shadow-lg` | Modals |
| `--shadow-glow` | Focus states |

### Typography

- **Headings**: System font stack, semibold
- **Body**: System font stack, regular
- **Monospace**: For code, IDs

## Responsive Breakpoints

| Breakpoint | Width | Layout Changes |
|------------|-------|----------------|
| Mobile | < 640px | Sidebar hidden, hamburger menu |
| Tablet | 640px - 1024px | Collapsed sidebar (icons only) |
| Desktop | > 1024px | Full sidebar on hover, multi-column |

---

## Interaction Patterns

1. **Hover States**: Subtle background changes, cursor pointer
2. **Focus States**: Ring outline with primary color glow
3. **Loading States**: Skeleton loaders, spinner for actions
4. **Error States**: Red border, error message below input
5. **Success Feedback**: Toast notifications (bottom-right)
6. **Dangerous Actions**: Confirmation dialogs with explicit warnings
